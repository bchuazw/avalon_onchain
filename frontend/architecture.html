üîê Security Model: Semi-Trusted Architecture

Role Assignment

Deterministic from VRF seed. Anyone can verify the algorithm produced correct distribution. Auditable by replaying with same seed.

On-Chain Verification

Merkle root stored in game account. Players prove role with Merkle proof. Smart contract validates proofs cryptographically.

Information Flow

Agents see only public state + own role. Spectators see full god view (read-only). Backend is role authority but cannot modify game state.

No Single Point of Failure

Game state is on-chain (immutable). Role assignment is deterministic (reproducible). Even if backend fails, on-chain state persists.
